問1　モバイル決済アプリのなりすまし対策とHMAC

設問1(1)
解答例
手段：他者のバーコードを会員番号から推測して表示する・他者の会員番号を窃取してバーコードを生成し決済する 問題：バーコードの内容が会員番号であること・バーコードが永続的に利用できること
バーコード決済のなりすまし脆弱性。連番や推測可能な会員番号をバーコードに使うと他者の番号を推測できる。また有効期限のない静的バーコードは一度盗撮されると永続的に悪用される。動的コード（一定時間で変わる）や HMAC値の埋め込みが対策。
設問1(2)
解答例
HMAC値αとHMAC値βの一致を検証する
HMAC（Hash-based Message Authentication Code）によるバーコードの完全性・真正性検証。共通鍵とハッシュ関数を組み合わせてMAC値を生成し、サーバ側で検証することでバーコードの改ざん・偽造を検知できる。

解答例
変更する設定項目：い（DNSサーバ設定） 変更後の設定内容：攻撃者のDNSサーバのIPアドレス
DHCP Starvation + Rogue DHCP攻撃。正規のDHCPサーバのアドレスプールを枯渇させ、偽のDHCPサーバから悪意あるDNSサーバのIPを配布することでDNSハイジャックを実行する。
