問1　バッファオーバフロー脆弱性と対策技術

設問1(1)
解答例
キ・ウ・ア（バッファオーバフローの仕組み）
スタックバッファオーバフロー攻撃の仕組み。バッファ境界を超えた書き込みでリターンアドレスを上書きし、攻撃者が用意したシェルコードにジャンプさせる。現代のCPU/OSは複数の軽減技術で防御する。
設問1(3)
解答例
シェルコードがDEPで実行禁止にされているスタック領域にあるから
DEP（Data Execution Prevention）/NX（No-eXecute）ビット。スタック・ヒープ等のデータ領域にあるコードを実行しようとするとCPUが例外を発生させる。シェルコードをスタックに配置する古典的攻撃を無効化する。
設問2(1)
解答例
e＝canary（スタックカナリア） f＝ASLR
軽減技術の組み合わせ。スタックカナリア（Canary）はリターンアドレス前にランダム値を配置しスタック破壊を検知する。ASLR（Address Space Layout Randomization）はメモリアドレスをランダム化し、攻撃コードの絶対アドレス特定を困難にする。
設問3
解答例
排除できない理由：ポインタを使って直接メモリ操作しているから 問題：メモリ破壊攻撃を防げないこと
C言語のポインタ操作はOS/CPU単位の保護を超えて任意メモリを読み書きできる。Use-After-Free・Heap Overflow等はカナリアやASLRだけでは防げず、言語レベルのメモリ安全性（RustやGo等の使用）やAddressSanitizer・Valgrindによる静的/動的解析が必要。
