問1　アジャイル開発環境のWebセキュリティ

設問1(1)
解答例
ア（オープンリダイレクト）
オープンリダイレクト脆弱性。URLパラメータで指定されたリダイレクト先を検証せずに遷移する実装で発生。フィッシングサイトへの誘導や認証後リダイレクトを悪用した攻撃に利用される。
設問1(2)
解答例
・ダウンロードするライブラリに既知の脆弱性がないかを確認する ・特定のWebサイトからの入手をルール化し明文化する
ソフトウェアサプライチェーンのセキュリティ対策。npm/pip等のパッケージ取得時に脆弱性スキャン（Snyk/Dependabot等）を実施し、非公式リポジトリからの取得を禁止するポリシーが重要。
設問2
解答例
a＝利用者ID　b＝セッションオブジェクト
セッション管理の設計。ユーザIDをCookieやURLに含めず、サーバ側のセッションオブジェクトに保持することで、ユーザID改ざんによる不正アクセスを防止する。
設問4
解答例
topic の値をhttps://db-y.b-sha.co.jp/に変更した（SSRFまたはオープンリダイレクト）
SSRF（Server-Side Request Forgery）またはオープンリダイレクトの悪用。topicパラメータで外部URLを指定することで、サーバを踏み台にして内部ネットワークリソースへのアクセスが可能になる。
設問5(1)
解答例
returnURLの値を固定値にする
オープンリダイレクト対策。リダイレクト先URLを動的に受け取らず、サーバ側で定義した許可リストのURLのみに制限する。またはリダイレクト先として有効なパス（/path形式）のみを受け付ける。
設問6(2)
解答例
・専門技術者による脆弱性診断が必要なときは改良リリースを次回に持ち越す ・定期的に期間の長い改良リリースを設ける
アジャイル開発でのセキュリティ診断の組み込み。スプリントの中で専門的な脆弱性診断を実施するための時間確保。セキュリティスプリントやセキュリティゲートを設けることで継続的なセキュリティ確保が可能。

問2　クラウドへの移行と認証・OIDC

設問1(1)
解答例
a＝キャッシュ（DNSキャッシュ）
CDNを利用したWebサイト高速化の仕組み。DNSキャッシュにより名前解決のレイテンシを削減し、エッジサーバのキャッシュにより静的コンテンツの配信を最適化する。
設問1(2)
解答例
b,c＝DDoS対応（CDN/Anycastによる）
CDNによるDDoS緩和。大容量のCDNインフラが攻撃トラフィックを吸収・分散し、オリジンサーバへの直撃を防ぐ。CDNのIPアドレスレート制限・WAFとの組み合わせが有効。
設問2(1)
解答例
STは認証サーバに送られないから
Kerberosプロトコルの仕組み。サービスチケット（ST）はリソースサーバに直接提示され、認証サーバ（KDC）を経由しない設計。TGT（チケット付与チケット）との違いを理解する。
設問2(2)
解答例
総当たり攻撃はオフラインで行われ、ログインに失敗しないから
オフラインブルートフォース攻撃の特性。パスワードハッシュが漏えいした場合、オフライン環境で総当たりを実施するためログイン失敗が記録されない。bcrypt等の低速ハッシュ関数でリスクを低減する。
